# fixtures/

Golden test fixtures exported from the real Python engine in
`yoruba-student-dict`, used by `shared/`'s Vitest suite to verify the
TypeScript port produces identical output to the original.

Generated by `yoruba-student-dict/scripts/export_js_port_fixtures.py`,
which runs the actual `generate_diagnostics.py`/`kaikki_search.py`/
`vocab_search.py`/`duplicate_check.py`/`syllabify.py`/
`yoruba_orthography.py` functions against the real vocab/lexicon/overrides
data and dumps input/output pairs - not just the aggregate
`dictionary_diagnostics.json`, but individually-named function-level cases,
specifically including every previously-fixed bug (the `owó`/`ọwọ́`
underdot conflation, the `ìlẹ̀`/`ilé` homograph gloss-pick, the
confirmed-vs-unconfirmed match-target distinction - see
`yoruba-student-dict/REMOTE_ACCESS_DISCUSSION.md` §4) as its own regression
fixture, not folded into a single whole-report diff.

Re-export and re-commit these whenever the Python engine's behavior
changes intentionally; a mismatch against `shared/`'s tests otherwise means
either the port drifted or the fixtures are stale - check which before
"fixing" either side.

Empty until `scripts/export_js_port_fixtures.py` is written and run.
